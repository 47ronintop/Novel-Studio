# Fixtures

Fixtures in this repository must be synthetic, small, and safe to commit.

Rules:

- Do not include real API keys, access tokens, provider credentials, or secrets.
- Do not include real user manuscripts or identifiable personal data.
- Prefer minimal valid and invalid examples that exercise one contract at a time.
- Large fixtures are allowed only for explicit performance tasks and must be documented.
- LLM fixtures must use mock responses or redacted provider samples.
