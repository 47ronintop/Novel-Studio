# M29 功能完成度盘点与命令面板执行闭环

版本：1.0 | 状态：M29 已采纳 | 阶段：Post-M18 产品化打磨

## 目标

M29 回应安装版试用时暴露出的核心问题：界面上仍有不少功能看起来存在，但真实完成度不一致。M29 不做欢迎页，也不包装尚未完成的能力；它先建立一份面向用户入口的完成度盘点，并补齐一个最影响“可点可用”感的入口：命令面板真实执行。

## 范围

M29 包含：

- 建立功能完成度审计表，将可见入口分为可用、半成品、仅提示、应补齐和后续路线。
- 将 M27 安装后首次使用引导从“下一步”下调为后续工作，避免先包装未完成功能。
- 命令面板命令列表从只读展示升级为可点击执行。
- 命令面板执行安全命令后同步 shell state，例如折叠导航、折叠检查器、切换底部面板。
- 命令面板继续只暴露 safe command，不新增文件系统访问、不新增真实模型调用、不新增 AI 自动写入。

M29 不包含：

- 欢迎页、示例项目和首次使用向导。
- 多 tab 编辑器。
- 底部面板真实多 tab 内容。
- 插件管理 UI 或插件市场。
- 时间线可视化画布。
- 自动保存和崩溃恢复 UI。

## 命令面板行为

- 命令面板打开后，safe command 以按钮形式展示。
- 用户点击命令后，renderer 调用现有 `NovelStudioApi.commands.execute(commandId)`。
- 执行成功后，renderer 用返回的 `DesktopShellState` 更新 UI。
- 执行失败后，不泄露底层异常或敏感信息；当前命令面板范围内不展示错误详情，后续可统一接入通知系统。
- `workspace.open-command-palette` 不应造成重复副作用；点击其它命令后命令面板关闭。

## 验收标准

- `CommandPalette` 测试覆盖命令项是按钮，并点击后回传命令 ID。
- Renderer 桥接测试覆盖点击命令后调用 `commands.execute` 并同步 shell state。
- Application 层仍只允许 safe command，未知命令继续返回脱敏 `APPLICATION_COMMAND_NOT_ALLOWED`。
- M29 文档、路线图、索引、changelog 和 beta readiness 同步当前状态。
- `format`、`typecheck`、`lint`、单元测试、contract、E2E、release 和 packaging gate 通过。
- 不 push；如需推送，必须先由用户确认。
