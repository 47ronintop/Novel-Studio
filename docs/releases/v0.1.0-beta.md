# Novel Studio v0.1.0 Beta

## M17 安装器与发布通道

- 新增 Windows NSIS installer target，同时保留 unpacked directory artifact，便于本地检查和 artifact secret scan。
- 新增本地 beta release channel manifest，并通过结构化 JSON Schema 校验。
- 新增 release notes 生成流程和 release check gate，覆盖 packaging metadata、installer settings、icon asset 与 signing policy。
- beta 发布保持手动流程；release scripts 不执行 push、上传或远程发布。
- 明确签名策略：M17 本地 beta artifact 允许未签名，未来证书签名通过 `CSC_LINK` 与 `CSC_KEY_PASSWORD` 接入。

## M18 插件系统边界

- 新增 schema-first 插件 manifest 与项目级 plugin registry，插件配置通过 contract tests 校验。
- 新增 `@novel-studio/plugin-engine`，提供离线、确定性的插件注册表构建、兼容性检查、禁用状态拒绝、能力检查和权限 scope 授权。
- 新建项目会创建 `plugins/plugins.json` 默认注册表，保持 local-first 项目结构一致。
- 插件引擎保持纯策略边界，不读写项目文件、不执行第三方代码、不访问 Electron、LLM、Repository、UI 或其他 Engine。
- M18 不引入真实插件市场、自动下载、网络访问或任意第三方代码执行。

## Beta 启动修复

- 安装版启动不再依赖源码 `fixtures/` 目录。
- 首次启动会在可写本地目录准备默认 beta 项目和首章，打开后直接显示可编辑章节正文。
- 运行时 JSON Schema contracts 会随 `app.asar` 一起打包，避免默认项目创建在安装环境中失败。
- 新增 Electron E2E 覆盖无源码 fixture 的安装版启动场景。

## Beta 启动 UX 与错误反馈

- Open/Create project 支持系统文件夹选择器，不再要求普通用户手动输入项目路径。
- 项目打开、创建和取消操作会在 Navigator 内显示明确状态或错误信息。
- Open/Create/Chapter 操作期间按钮会显示 busy 状态并临时禁用，避免重复点击造成不确定结果。
- Renderer 仍只通过 preload API 调用 allowlisted IPC，文件系统和系统对话框访问保留在 Electron main 边界。

## M19 Beta UX 产品化打磨

- 左侧 Activity Bar 六个入口可点击，并能切换工作区、搜索、时间线、AI 工作流、创作系统和设置视图。
- 顶部 Electron 应用菜单本地化为中文：`文件`、`编辑`、`视图`、`窗口`、`帮助`。
- 工作区、命令面板、章节编辑器、模型设置和 Studio 面板的主要用户可见文案中文化；技术 ID、provider ID 和 schema 字段保持英文。
- Search、Timeline、Studio、Settings 等尚未完整产品化的入口显示中文空状态，不再表现为无响应按钮。
- 新增真实 Electron E2E 覆盖左侧 Activity Bar 点击切换。

## M20 搜索与索引体验

- 新增 schema-first 搜索索引，索引文件落在可重建的 `cache/indexes/search.json`。
- Search 视图支持输入关键词、重建索引、查看索引条目数和搜索结果。
- 搜索覆盖章节正文、人物、世界观、大纲、时间线和记忆；renderer 仍只通过 preload/IPC 访问 Application 层。

## M21 故事圣经编辑体验

- 左侧 Activity Bar 新增“故事圣经”入口。
- 主视图支持人物、世界观、大纲、时间线和记忆的基础编辑表单。
- 保存仍通过 Story Bible preload/API 边界执行，保存后刷新摘要和编辑状态。

## M22 设置体验补齐

- Settings 视图从空状态升级为正式设置页。
- 模型 profile 支持编辑、新建草稿、保存、设为默认和连接测试。
- 设置页补齐自动保存与历史、隐私与安全中文说明。
- 密钥仍只使用 `secret://` 引用，UI 不显示明文 API Key，CI 不访问真实模型 endpoint。

## M23 Studio 体验补齐

- Studio 视图从空状态升级为真实创作系统工作台。
- 新建项目默认包含默认审稿 Prompt、默认审稿 Agent 和审稿当前章节 Workflow，安装版打开后有可编辑配置资产。
- 工作台支持选择 Prompt/Agent/Workflow 资产、编辑 JSON、保存配置资产和查看可恢复版本历史。
- 无效 JSON 会在 renderer 阻止保存；schema 校验、历史快照和恢复仍通过 Application/Repository 边界执行。
- UI 不直接访问文件系统，不新增 AI 自动写入，不访问真实模型 endpoint。

## M24 工作流运行观测

- AI 写作建议结果新增运行观测数据，覆盖 workflow run、Context trace、模型 profile、token/cost 和步骤状态。
- Inspector 的 AI Workflow 面板新增“AI 工作流运行观测”，用户可看到上下文来源、模型、Token、成本和当前待确认步骤。
- Agent handoff 只暴露脱敏 metadata，不显示 `secret://`、明文 API Key 或底层 provider 敏感错误详情。
- AI 输出仍保持建议态，应用到正文前必须由用户确认。

## M25 工作流运行历史

- AI 写作建议生成成功后新增本地 workflow run history，落点为 `history/workflows/runs/`，不进入可清理 cache。
- 新增 `workflow-run-record` schema，运行历史只保存脱敏 workflow、context、model、usage/cost 和 step 摘要。
- Desktop IPC/preload 支持列出和读取运行历史，Renderer 不直接访问文件系统。
- AI Workflow 面板新增“工作流运行历史”，展示最近运行、状态、模型、token/cost 和选中运行步骤详情。

## M26 工作流失败诊断与重试策略

- AI 写作工作流在模型/Agent 失败后写入本地 `failed` workflow run history。
- 失败记录只保存脱敏错误 code、message、recoverability、suggestedAction、retryable、retry policy 和步骤状态。
- AI Workflow 面板新增“失败诊断”和“重试策略”，并提供“重试 AI 工作流”按钮。
- 重试由用户主动触发，复用当前指令重新生成；成功输出仍保持建议态，应用到正文前必须由用户确认。
- 不显示明文密钥、完整 prompt、完整章节正文、provider raw payload 或未脱敏底层异常。

## M28 全局功能可用性盘点

- 标题栏“命令面板”按钮补齐可访问名称和 tooltip，并接入打开命令面板行为。
- 当前唯一打开资产的编辑器 tab 明确禁用并说明多 tab 切换将在后续补齐。
- 底部面板 tabs 明确禁用并说明面板切换将在后续补齐。
- 高可见入口不再表现为无反馈按钮；尚未实现的入口使用中文说明当前状态。
- M27 安装后首次使用引导尚未包含在本 beta 线中，后续需回补。

## M29 功能完成度盘点与命令面板执行闭环

- 新增功能完成度审计，明确工作区、搜索、故事圣经、时间线、设置、Studio、命令面板、底部面板和插件 UI 的真实状态。
- 按用户反馈暂缓 M27 欢迎引导，优先补齐核心功能缺口。
- 命令面板 safe command 从只读展示升级为可点击执行。
- 执行命令后 renderer 通过 preload API 同步 `DesktopShellState`，导航、检查器和底部面板切换不再停留在展示层。
- M29 不新增真实模型调用，不新增文件系统直接访问，也不引入 AI 自动写入。

## M30 底部面板工作区

- 底部面板 tabs 从禁用状态升级为可点击切换。
- 新增工作流运行、问题、搜索和日志四个最小内容面板。
- 工作流面板展示当前 AI 工作流状态、最近运行数量和失败诊断摘要。
- 搜索面板展示索引条目、当前查询和结果数量。
- 日志面板明确本地 beta 不采集 telemetry，默认流程不访问真实模型 endpoint。
- 切换底部 tab 只更新 renderer 内存 shell state，不写项目文件。

## 包含的里程碑

- M0-M26、M28-M30 包含在当前 beta 线中；M27 已暂缓。
- 当前 beta 仍不新增 telemetry，不引入真实模型调用，也不执行远程发布。
