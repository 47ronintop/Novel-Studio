# 设置页真实功能化与编辑器状态栏收敛设计

**日期：** 2026-07-10  
**状态：** 已确认  
**范围：** Novel Studio 桌面端工作台设置模式、设置页、章节编辑器、普通文件编辑器与底部状态栏

## 1. 目标

本轮解决三个直接影响可信度的问题：

1. 设置页成为工作台级独立视图，不再作为普通内容塞进中间编辑器区域。
2. 设置页中只保留真实可操作并会影响应用行为的条目；必要说明必须紧邻相关控件，不再以独立分类或展示区充当功能。
3. 编辑器顶部恢复为安静的 IDE 工作区，只保留真实打开文档及高频操作；查找替换按需浮动，文档状态与统计信息统一进入窗口底部状态栏。

完成后，页面上每个可见条目必须属于以下两类之一：

- 可以操作，并通过现有应用边界读写真实状态。
- 为理解相邻控件或当前状态所必需的说明。

## 2. 非目标

本轮不实现以下能力：

- 自动保存间隔、历史快照策略和恢复策略的编辑 UI。
- 把 `settings.json` 中尚未驱动运行时的 autosave/history 字段包装成表面可用的控件。
- 独立的关于、诊断或 Runtime 信息页。
- 新的插件安装、下载或权限授予流程。
- 重构编辑器 Runtime 适配器边界。
- 新建独立 Electron BrowserWindow 承载设置。

## 3. 工作台级设置模式

设置不再通过 `ActivityEmptyState` 渲染到 `.ns-editor-area`。当 `activeActivity === "settings"` 时，Workspace Shell 进入显式设置分支，顶栏下方的全部可用空间由独立 `SettingsWorkspace` 占据。

设置模式保留：

- 窗口顶栏。
- 项目名称。
- 全局命令面板入口和命令面板本身。

设置模式隐藏且不创建以下工作区区域：

- 活动栏。
- 项目 Navigator 与 resize handle。
- 编辑器标签和编辑器正文。
- AI 面板与 resize handle。
- 底部面板。
- 编辑器状态栏。
- 只作用于编辑器布局的底部面板、Split View 和专注模式按钮。

显式分支优先于只用 CSS 隐藏，因为不可见的编辑器控件不应残留在键盘焦点顺序或辅助技术树中。

`SettingsWorkspace` 自身包含：

- 左上设置标题。
- 设置搜索。
- 设置分类导航。
- 设置内容区域。
- 右上关闭图标按钮，带 tooltip 和 accessible label。

进入设置前记录最后一个非设置 Activity。关闭按钮或 `Escape` 返回该 Activity，而不是固定返回工作区首页。设置模式只改变当前呈现层级，不销毁章节草稿、选区、打开标签、面板尺寸或底部面板状态。

窄窗口下，设置分类导航从左侧纵向列表变为内容顶部的紧凑标签栏，设置内容保持单列并允许纵向滚动。

## 4. 设置页信息架构

设置页左侧仅保留三个分类：

- 模型
- 外观
- 插件

删除以下分类和展示块：

- 编辑器：其字体、字号和行高并入外观，避免重复入口。
- 写作：当前只展示读取结果，且 autosave/history 配置未完整驱动运行时。
- 高级：当前只有隐私与安全说明，没有独立操作。
- 首页中的“自动保存与历史”承诺性说明。
- 外观预览及其状态徽章。
- 未生效的界面密度设置。

API Key 的安全说明保留在模型表单的密钥字段旁。它解释真实存储行为，不再占用一个看似独立功能的设置分类。

设置页标题说明同步改为真实范围，只描述模型、外观和插件管理。

## 5. 外观设置

外观设置提供以下真实控件：

### 5.1 主题

主题值扩展为：

- `dark`
- `light`
- `system`

选择后立即作用于整个工作台，并通过现有用户偏好 API 持久化。`system` 使用操作系统颜色偏好决定实际主题。

### 5.2 强调色

新增受控的强调色枚举：

- `teal`：保持当前产品识别色，作为默认值。
- `blue`：更接近传统 IDE 的冷色强调。
- `amber`：为长时间写作提供暖色但克制的选择。

界面使用真实色板控件表达颜色，选中状态具备可访问标签和清晰焦点样式。强调色只改变主操作、选中态和焦点等语义色，不改变错误、警告、成功等状态色。

### 5.3 编辑器排版

保留并复用现有真实偏好：

- 字体：等宽、衬线、无衬线。
- 字号：限制在现有安全范围内。
- 行高：限制在现有安全范围内。

这些偏好继续同时作用于章节编辑器和普通文件编辑器。

## 6. 偏好数据流与兼容

`UserAppearancePreferences` 调整为主题与强调色，不再对外暴露界面密度：

```ts
interface UserAppearancePreferences {
  readonly theme: "dark" | "light" | "system";
  readonly accentColor: "teal" | "blue" | "amber";
}
```

数据流保持现有边界：

```text
外观控件
  -> App 本地状态立即更新
  -> NovelStudioApi.preferences.save
  -> UserPreferencesSession 归一化
  -> UserPreferencesFileRepository 原子写入
  -> 下次启动 preferences.load 恢复
```

Renderer 根据偏好在文档根节点设置主题和强调色数据属性，CSS token 根据属性切换。系统主题通过 `prefers-color-scheme` 生效。

兼容规则：

- 旧文件缺少 `accentColor` 时归一化为 `teal`。
- 旧文件中的 `density` 允许被读取，但归一化后的新快照不再依赖它。
- 未知主题或强调色回退到 `dark` 与 `teal`。
- 继续使用现有 schemaVersion，不引入与本轮规模不相称的迁移框架。

偏好保存采用即时应用。保存失败时，设置页显示内联错误，明确说明当前选择只在本次会话生效；不得显示误导性的成功状态。

设置工作台切换不进入偏好文件。Renderer 在进入设置时保存最近的非设置 Activity，并在关闭时通过现有 Activity 切换入口恢复。应用层继续把 `settings` 作为合法 Activity id，但 UI 不再把它当作编辑器内容类型。

## 7. 编辑器文档栏

编辑器顶部只保留一条 VSCode 式文档栏，不再在标签栏下方增加固定工具栏。编辑器内部删除重复的章节标题、保存状态、行数、字符数和 Runtime 徽章区。

文档栏左侧只显示真实打开文档：

- 章节使用 `${chapter.title}.md` 作为可读的文件式显示名，底层仍使用稳定章节 ID 和现有磁盘路径。
- 普通文件显示真实文件名。
- 从 Navigator 打开文档时新增或激活对应标签。
- 标签数据只来自显式打开文档状态；`openChapterTabIds` 缺失或为空时不得回退为全部章节。
- 活动标签必须与当前正文一致。
- 未保存文档显示 dirty 圆点；关闭按钮在活动或 hover 状态出现。
- 切换或关闭标签继续使用现有回调，不在 UI 层直接修改项目文件。

文档栏右侧只包含当前文档真正可用的高频操作：

- 保存。
- 查找或替换。
- 专注模式。
- 选区 AI 预览，仅在存在有效选区且回调可用时出现。

按钮使用图标、tooltip 和 accessible label。没有对应回调的操作不渲染，不显示一个看似可用但点击无效的按钮。

## 8. 查找与替换

查找替换不占据固定顶部行，也不推动正文布局。它作为编辑区右上角的按需浮层出现，定位范围限制在当前编辑器表面内。

打开方式：

- 点击文档栏的查找按钮，打开查找模式。
- `Ctrl/Cmd+F` 打开查找模式并聚焦查找输入框。
- `Ctrl/Cmd+H` 打开替换模式并显示替换行。

浮层结构：

- 第一行包含查找输入、匹配计数、上一处、下一处、区分大小写、展开替换和关闭按钮。
- 第二行仅在替换模式下出现，包含替换输入、替换当前和全部替换。
- 查找或替换动作继续复用现有纯函数逻辑，避免重写匹配算法。

交互规则：

- `Escape` 关闭浮层，并把焦点还给当前编辑器。
- 关闭按钮执行同样的焦点恢复。
- 切换文档标签时关闭查找浮层，避免查询状态错误地跨文档悬挂。
- 浮层打开时不会遮住文档栏；正文可滚动区域保留足够的右上内边距，避免输入内容完全被覆盖。
- 窄窗口下浮层宽度受编辑器可用宽度限制，内部控件保持两行布局，不横向溢出。
- 章节和普通文件编辑器使用同一个查找替换组件与交互模型；普通文件不再显示“普通文件模式”占位条。

Runtime 警告只有在真实警告存在时才显示为警告区域。以下开发信息不进入常规编辑界面：

- CodeMirror / Textarea 适配器名称
- 默认快捷键配置
- Runtime 迁移门禁
- 本地 diff Runtime 摘要
- 无操作意义的技术状态徽章

## 9. 底部状态栏

复用现有 `.ns-status-bar`，调整为左右分组的紧凑布局。

左侧显示：

- 当前文档保存状态。
- 恢复草稿可用状态，仅在真实存在时显示。

右侧显示：

- 字数。
- 预计阅读时间。
- 当前光标行列；有选区时显示选区起止行或选中字数。
- 当前文档模式，例如 Markdown。

状态栏不再显示章节标题和 AI 工作流状态：标题已存在于标签中，AI 状态不属于文档编辑状态。

章节数据来自 `ChapterEditorProps` 与 Runtime selection snapshot；普通文件数据来自 `PlainFileEditorProps` 和编辑器本地选择状态。字数与阅读时间继续复用 `calculateWritingMetrics`，避免重复算法。

没有打开文档时，状态栏不伪造 Markdown、字数或行列信息，只显示存在的工作区状态。

设置模式不显示编辑器状态栏，因为此时没有可见的编辑文档，继续显示 Markdown、字数或行列会形成错误上下文。

## 10. 错误与边界状态

- 外观偏好保存失败：显示内联错误；当前会话样式可以保留，但明确未持久化。
- 系统主题不可解析：回退深色主题。
- 未知强调色：回退 `teal`。
- 没有当前编辑器：隐藏文档统计项。
- Runtime 没有选择快照：行列显示文档起始位置或由当前编辑器本地状态提供。
- 普通文件缺少正文更新回调时，查找仍可用但替换操作禁用，并显示明确 disabled 状态。
- 保存按钮在无修改或保存中保持禁用，并保留清晰的 tooltip/label。
- 没有可恢复的前一个 Activity 时，关闭设置回到 `workspace`。
- 设置数据尚未加载时，独立设置视图显示真实加载或不可用状态，不回退成普通编辑器空态。
- 没有打开文档时不渲染空标签或伪造的“未命名章节”标签。

## 11. 测试与验收

### 11.1 自动化测试

更新或新增以下覆盖：

- 用户偏好默认值包含 `accentColor`。
- `light`、`dark`、`system` 与强调色能保存和恢复。
- 旧偏好缺少强调色或包含旧 density 时可以归一化。
- 设置导航只包含模型、外观、插件。
- 设置页不再渲染写作、高级、自动保存说明、密度和外观预览。
- 外观控件触发真实回调并使用可访问的主题/色板控件。
- 章节编辑器顶部不渲染 Runtime、统计徽章、重复标题区或第二条固定工具栏。
- 文档栏只呈现显式打开的章节和普通文件，不回退显示全部章节。
- 章节标签显示 `章节标题.md`，普通文件标签显示真实文件名。
- 文档栏操作只呈现有回调的命令。
- 查找按钮、`Ctrl/Cmd+F` 和 `Ctrl/Cmd+H` 能按正确模式打开浮层。
- 查找浮层包含关闭按钮，`Escape` 和关闭按钮都会关闭浮层并恢复编辑器焦点。
- 查找浮层不改变编辑器布局高度，替换行只在替换模式出现。
- 切换标签时关闭查找浮层。
- 普通文件编辑器复用真实查找替换组件，不渲染占位条。
- 状态栏呈现真实保存状态、字数、阅读时间、行列与文档模式。
- 没有文档时不呈现伪造的文档状态。
- Runtime 警告仅在警告数组非空时渲染。
- 设置 Activity 使用独立工作台分支，不通过编辑区的 `ActivityEmptyState` 渲染。
- 设置模式不渲染活动栏、Navigator、编辑器标签、AI 面板、底部面板和状态栏。
- 设置模式保留全局命令面板入口并隐藏编辑器布局控制。
- 关闭按钮和 `Escape` 返回进入设置前的非设置 Activity。
- 进入和退出设置不会修改草稿、打开标签与布局状态。

### 11.3 视觉验收

在桌面窗口和窄窗口分别检查：

- 设置页占据顶栏以下的完整工作台区域，不夹在 Navigator 与 AI 面板之间。
- 设置模式没有编辑器标签、底部状态栏或无关布局按钮残留。
- 关闭设置后返回原 Activity，原编辑器内容和布局保持不变。
- 窄窗口的设置分类导航切换为顶部标签，内容不被压缩成狭窄中栏。
- 编辑器正文上方只有单条文档栏，没有状态徽章堆叠或第二条工具栏。
- 标签名称对应实际打开文档，章节名称具备 `.md` 后缀。
- 查找替换关闭时完全不占空间；打开时浮在正文右上角且不推动布局。
- 查找浮层、文档栏和正文之间没有不可读的遮挡。
- 状态栏始终贴合窗口底部，字号小且不抢正文焦点。
- 最长中文标签和错误文案不会溢出控件。
- 深色、浅色和系统主题均具备可读对比度。
- 三种强调色不会改变错误、警告和成功语义。

## 12. 完成定义

本设计完成时：

- 设置页所有可见设置都真实生效并可持久化。
- 设置页不存在承诺未来能力的占位说明。
- 设置作为工作台级独立视图打开和关闭，不再占用普通编辑器内容区域。
- 编辑器顶部只保留真实打开文档与高频命令组成的单条文档栏。
- 查找替换仅按需浮动出现，章节与普通文件共享同一套完整交互。
- 技术性和状态性信息进入底部状态栏或从常规界面移除。
- 章节与普通文件编辑体验使用同一套信息层级。
- 自动化验证通过，并完成桌面与窄窗口视觉检查。
