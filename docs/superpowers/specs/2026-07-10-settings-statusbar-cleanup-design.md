# 设置页真实功能化与编辑器状态栏收敛设计

**日期：** 2026-07-10  
**状态：** 已确认  
**范围：** Novel Studio 桌面端设置页、章节编辑器、普通文件编辑器与底部状态栏

## 1. 目标

本轮解决两个直接影响可信度的问题：

1. 设置页中只保留真实可操作并会影响应用行为的条目；必要说明必须紧邻相关控件，不再以独立分类或展示区充当功能。
2. 编辑器顶部恢复为安静的 IDE 工作区，只保留标签和高频操作；文档状态与统计信息统一进入窗口底部状态栏。

完成后，页面上每个可见条目必须属于以下两类之一：

- 可以操作，并通过现有应用边界读写真实状态。
- 为理解相邻控件或当前状态所必需的说明。

## 2. 非目标

本轮不实现以下能力：

- 自动保存间隔、历史快照策略和恢复策略的编辑 UI。
- 把 `settings.json` 中尚未驱动运行时的 autosave/history 字段包装成表面可用的控件。
- 独立的关于、诊断或 Runtime 信息页。
- 新的插件安装、下载或权限授予流程。
- 重构编辑器 Runtime 适配器边界。

## 3. 设置页信息架构

设置页左侧仅保留三个分类：

- 模型
- 外观
- 插件

删除以下分类和展示块：

- 编辑器：其字体、字号和行高并入外观，避免重复入口。
- 写作：当前只展示读取结果，且 autosave/history 配置未完整驱动运行时。
- 高级：当前只有隐私与安全说明，没有独立操作。
- 首页中的“自动保存与历史”承诺性说明。
- 外观预览及其状态徽章。
- 未生效的界面密度设置。

API Key 的安全说明保留在模型表单的密钥字段旁。它解释真实存储行为，不再占用一个看似独立功能的设置分类。

设置页标题说明同步改为真实范围，只描述模型、外观和插件管理。

## 4. 外观设置

外观设置提供以下真实控件：

### 4.1 主题

主题值扩展为：

- `dark`
- `light`
- `system`

选择后立即作用于整个工作台，并通过现有用户偏好 API 持久化。`system` 使用操作系统颜色偏好决定实际主题。

### 4.2 强调色

新增受控的强调色枚举：

- `teal`：保持当前产品识别色，作为默认值。
- `blue`：更接近传统 IDE 的冷色强调。
- `amber`：为长时间写作提供暖色但克制的选择。

界面使用真实色板控件表达颜色，选中状态具备可访问标签和清晰焦点样式。强调色只改变主操作、选中态和焦点等语义色，不改变错误、警告、成功等状态色。

### 4.3 编辑器排版

保留并复用现有真实偏好：

- 字体：等宽、衬线、无衬线。
- 字号：限制在现有安全范围内。
- 行高：限制在现有安全范围内。

这些偏好继续同时作用于章节编辑器和普通文件编辑器。

## 5. 偏好数据流与兼容

`UserAppearancePreferences` 调整为主题与强调色，不再对外暴露界面密度：

```ts
interface UserAppearancePreferences {
  readonly theme: "dark" | "light" | "system";
  readonly accentColor: "teal" | "blue" | "amber";
}
```

数据流保持现有边界：

```text
外观控件
  -> App 本地状态立即更新
  -> NovelStudioApi.preferences.save
  -> UserPreferencesSession 归一化
  -> UserPreferencesFileRepository 原子写入
  -> 下次启动 preferences.load 恢复
```

Renderer 根据偏好在文档根节点设置主题和强调色数据属性，CSS token 根据属性切换。系统主题通过 `prefers-color-scheme` 生效。

兼容规则：

- 旧文件缺少 `accentColor` 时归一化为 `teal`。
- 旧文件中的 `density` 允许被读取，但归一化后的新快照不再依赖它。
- 未知主题或强调色回退到 `dark` 与 `teal`。
- 继续使用现有 schemaVersion，不引入与本轮规模不相称的迁移框架。

偏好保存采用即时应用。保存失败时，设置页显示内联错误，明确说明当前选择只在本次会话生效；不得显示误导性的成功状态。

## 6. 编辑器顶部结构

工作区顶部保留现有章节标签栏。编辑器内部删除重复的标题、保存状态、行数、字符数和 Runtime 徽章区。

紧凑工具栏只包含真实高频操作：

- 保存
- 查找替换
- 专注模式
- 选区 AI 预览，仅在存在有效选区且回调可用时出现

按钮使用图标、tooltip 和 accessible label。查找替换面板仍在打开时出现在编辑器上方，因为它是当前任务的一部分。

Runtime 警告只有在真实警告存在时才显示为警告区域。以下开发信息不进入常规编辑界面：

- CodeMirror / Textarea 适配器名称
- 默认快捷键配置
- Runtime 迁移门禁
- 本地 diff Runtime 摘要
- 无操作意义的技术状态徽章

## 7. 底部状态栏

复用现有 `.ns-status-bar`，调整为左右分组的紧凑布局。

左侧显示：

- 当前文档保存状态。
- 恢复草稿可用状态，仅在真实存在时显示。

右侧显示：

- 字数。
- 预计阅读时间。
- 当前光标行列；有选区时显示选区起止行或选中字数。
- 当前文档模式，例如 Markdown。

状态栏不再显示章节标题和 AI 工作流状态：标题已存在于标签中，AI 状态不属于文档编辑状态。

章节数据来自 `ChapterEditorProps` 与 Runtime selection snapshot；普通文件数据来自 `PlainFileEditorProps` 和编辑器本地选择状态。字数与阅读时间继续复用 `calculateWritingMetrics`，避免重复算法。

没有打开文档时，状态栏不伪造 Markdown、字数或行列信息，只显示存在的工作区状态。

## 8. 错误与边界状态

- 外观偏好保存失败：显示内联错误；当前会话样式可以保留，但明确未持久化。
- 系统主题不可解析：回退深色主题。
- 未知强调色：回退 `teal`。
- 没有当前编辑器：隐藏文档统计项。
- Runtime 没有选择快照：行列显示文档起始位置或由当前编辑器本地状态提供。
- 普通文件查找替换尚不可用时，不显示一个无回调的查找按钮。
- 保存按钮在无修改或保存中保持禁用，并保留清晰的 tooltip/label。

## 9. 测试与验收

### 9.1 自动化测试

更新或新增以下覆盖：

- 用户偏好默认值包含 `accentColor`。
- `light`、`dark`、`system` 与强调色能保存和恢复。
- 旧偏好缺少强调色或包含旧 density 时可以归一化。
- 设置导航只包含模型、外观、插件。
- 设置页不再渲染写作、高级、自动保存说明、密度和外观预览。
- 外观控件触发真实回调并使用可访问的主题/色板控件。
- 章节编辑器顶部不渲染 Runtime、统计徽章和重复标题区。
- 工具栏只呈现有回调的操作。
- 状态栏呈现真实保存状态、字数、阅读时间、行列与文档模式。
- 没有文档时不呈现伪造的文档状态。
- Runtime 警告仅在警告数组非空时渲染。

### 9.3 视觉验收

在桌面窗口和窄窗口分别检查：

- 编辑器正文上方没有状态徽章堆叠。
- 标签、工具栏、查找替换和正文之间没有重叠。
- 状态栏始终贴合窗口底部，字号小且不抢正文焦点。
- 最长中文标签和错误文案不会溢出控件。
- 深色、浅色和系统主题均具备可读对比度。
- 三种强调色不会改变错误、警告和成功语义。

## 10. 完成定义

本设计完成时：

- 设置页所有可见设置都真实生效并可持久化。
- 设置页不存在承诺未来能力的占位说明。
- 编辑器顶部只保留标签、高频命令和按需出现的查找/警告区域。
- 技术性和状态性信息进入底部状态栏或从常规界面移除。
- 章节与普通文件编辑体验使用同一套信息层级。
- 自动化验证通过，并完成桌面与窄窗口视觉检查。
