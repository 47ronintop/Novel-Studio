# M72-M74 Runtime Preview Design

## Scope

M72-M74 addresses the next runtime productization batch:

- M72 adds a fixture-backed plugin sandbox worker adapter that enforces timeout and max output bytes before normal plugin runtime output normalization.
- M73 adds a CodeMirror DOM mount planning contract so future UI code can request a DOM view mount through the renderer adapter boundary.
- M74 adds Application-backed selection-aware AI preview generation and routes it through IPC/preload/renderer bridge APIs as preview-only diff data.

## Architecture

Plugin sandbox fixture execution remains in the Application layer. `PluginRuntimeSession` continues to validate plugin enablement, contribution existence, capabilities, and granted permissions before any adapter call. The new fixture worker adapter is deterministic test infrastructure for the sandbox boundary; it accepts fixture outputs, duration metadata, timeout limits, and payload limits. It never loads plugin files or executes arbitrary code.

CodeMirror DOM mount planning remains inside the renderer editor runtime module. The adapter can report a structured DOM mount plan when explicitly mounted with a DOM target descriptor. The plan records package intent and fallback state, but the default runtime remains textarea and no storage/model/Electron API is touched.

Selection-aware AI preview is generated by `AiWritingWorkflowSession`. Renderer code sends a structured selection request through `NovelStudioApi`, preload, IPC, and `DesktopApplication`; Application invokes the existing Agent/LLM boundary and returns `previewOnly` diff data. The bridge displays that diff through the existing AI workflow panel and chapter diff preview, but does not set an applyable suggestion id.

## Data Flow

Plugin fixture worker:

`PluginRuntimeSession.runWorkflowStep()` -> policy validation -> `PluginRuntimeAdapter.executeWorkflowStep()` -> fixture worker timeout/output limits -> structured output normalization.

CodeMirror mount plan:

`createCodeMirrorEditorRuntimeAdapter().mount({ domMountTarget })` -> `EditorRuntimeSnapshot.domViewMount` -> runtime status/debug surfaces.

Selection AI preview:

`EditorSelectionCommand` -> renderer AI bridge -> `NovelStudioApi.ai.generateSelectionPreview()` -> IPC -> `DesktopApplication.generateActiveSelectionPreview()` -> `AiWritingWorkflowSession.generateSelectionPreview()` -> LLM Adapter -> preview-only diff.

## Non-goals

- No arbitrary third-party plugin execution.
- No real sandbox process isolation or marketplace signing prompts.
- No default CodeMirror switch or automatic DOM view replacement.
- No renderer-side model calls.
- No automatic selection replacement.

## Testing

- Application tests verify fixture worker timeout and max output bytes.
- Renderer runtime tests verify CodeMirror DOM mount plan metadata and textarea fallback.
- Application AI workflow tests verify selection preview output, model request payload, and no chapter writes.
- Desktop IPC/preload/bridge tests verify selection preview routes only through Application channels.
